Как устроена система событий React? К чему привязываются слушатели и что изменилось в React 17? Как всплывают события из портала и почему это иногда удивляет? Чем синтетическое событие отличается от нативного?

<!-- answer -->

**Синтетические события.** React не вешает слушатель на каждый `<button onClick>`. Вместо этого он использует **делегирование**: ставит небольшое число слушателей на корневой контейнер и получает события всплытием, а затем создаёт кросс-браузерную обёртку `SyntheticEvent` и прогоняет её по дереву **компонентов**. Плюсы: меньше слушателей (важно для больших списков), единый нормализованный API, контроль над порядком.

**Что изменилось в React 17.** Раньше React вешал делегирование на `document`. С 17-й версии — на **корневой DOM-узел приложения** (тот, что передан в `createRoot`). Зачем: чтобы на одной странице могли сосуществовать несколько независимых версий React или React рядом с другим фреймворком — их события не пересекаются, а `e.stopPropagation()` на промежуточном узле реально останавливает распространение (при делегировании на document это работало неочевидно). Также это упростило постепенную миграцию.

**Порталы — главный сюрприз.** `createPortal(children, domNode)` рендерит DOM ребёнка в **другом месте дерева DOM** (модалка в `body`), но в **дереве React** ребёнок остаётся под родителем. События всплывают **по дереву React, а не по DOM**:

```jsx
function Modal({ children }) {
  return createPortal(<div className="modal">{children}</div>, document.body);
}
// <Layout onClick={...}> ловит клики из модалки,
// хотя её DOM живёт в body, вне Layout
```

Поэтому `onClick` на родителе-обёртке поймает клик из модалки, отрендеренной в `body` — что удобно (общий обработчик, контекст работает), но ломает интуицию «раз в DOM снаружи — значит не всплывёт сюда». Учитывайте при «клик вне для закрытия»: нативный `document`-листенер не увидит логической вложенности.

**Отличия от нативных событий:**

- **Капчур**: React поддерживает фазу перехвата через `onClickCapture` и т.п.
- **Пул событий убран** (был до React 17): раньше `SyntheticEvent` переиспользовался и обнулялся после обработчика — асинхронный доступ к `e` требовал `e.persist()`. Сейчас пула нет, событие можно читать асинхронно.
- **passive-слушатели**: React вешает скролл/тач-события пассивно там, где это выигрышно для производительности — поэтому `preventDefault()` в `onWheel`/`onTouchMove` может не сработать; для таких случаев нужен нативный `addEventListener` через ref с `{ passive: false }`.
- Доступ к нативному событию — через `e.nativeEvent`.

**Практика:** для «клика снаружи», hotkeys на уровне окна, ResizeObserver, non-passive preventDefault — вешайте нативные слушатели через ref/эффект; для обычного UI — синтетические `onX`. Помните про порталы, когда рассуждаете о всплытии и «внешних» кликах.
