Для чего нужны рефы и чем они отличаются от состояния? Объясните объектные и callback-рефы, роль `forwardRef` и `useImperativeHandle`. Когда императивный доступ оправдан, а когда это запах?

<!-- answer -->

**Ref vs state.** Оба «переживают» ререндеры, но `useState` — часть модели рендера: изменение планирует перерисовку, значение стабильно в течение рендера. `useRef` — «карман» вне рендера: `ref.current` можно менять в любой момент, это **не** вызывает ререндер и **не** отражается в JSX синхронно. Правило: если значение влияет на то, что видит пользователь — это state; если это «побочный» дескриптор (DOM-узел, id таймера, предыдущее значение, флаг «уже инициализировано») — это ref.

```jsx
const inputRef = useRef(null);
// <input ref={inputRef} />
const focus = () => inputRef.current?.focus(); // императивный доступ к DOM
```

Нельзя читать/писать `ref.current` **во время рендера** (кроме ленивой инициализации) — рендер должен быть чистым; работа с ref — в обработчиках и эффектах.

**Объектный vs callback ref.** Объектный (`useRef`) React кладёт узел в `.current`. **Callback ref** — функция `(node) => {...}`: React зовёт её с узлом при монтировании и с `null` при размонтировании. Нужен, когда надо *отреагировать* на появление/исчезновение узла (измерить размер, подписаться, собрать список динамических рефов). В React 19 из callback ref можно вернуть функцию-cleanup, как в эффекте.

**forwardRef и его судьба.** Ref не передаётся как обычный проп. Исторически, чтобы родитель дотянулся до DOM внутри компонента, компонент оборачивали в `forwardRef((props, ref) => ...)`. **В React 19** `ref` доступен как обычный проп функционального компонента — `forwardRef` больше не нужен для новых компонентов (оставлен для совместимости):

```jsx
function Input({ ref, ...props }) { // React 19: ref как проп
  return <input ref={ref} {...props} />;
}
```

**useImperativeHandle** позволяет отдать наружу **не сам DOM, а кастомный императивный API**:

```jsx
function Modal({ ref }) {
  const dialog = useRef(null);
  useImperativeHandle(ref, () => ({
    open: () => dialog.current.showModal(),
    close: () => dialog.current.close(),
  }), []);
  return <dialog ref={dialog}>...</dialog>;
}
// parent: modalRef.current.open()
```

Так скрывают внутренности и дают узкий контракт вместо голого узла.

**Когда это запах.** Рефы и императив — «аварийный люк» из декларативной модели. Оправдано: фокус/выделение, скролл, media-элементы, интеграция с не-React библиотеками (карты, canvas, редакторы), измерения. Запах: управлять через ref тем, что должно быть state (открыт/закрыт, значение поля) — это рассинхронизирует UI с данными. Сначала ищите декларативное решение (проп, состояние, key), ref — когда его нет.
