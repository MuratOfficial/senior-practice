Что такое замыкание? Объясните механизм на уровне лексических окружений. Приведите пример, когда замыкание приводит к утечке памяти, и как это чинить.

<!-- answer -->

**Замыкание** — функция вместе с её лексическим окружением: она сохраняет доступ к переменным области, где была **создана**, даже когда та область уже завершила выполнение.

Механизм: при создании функция получает скрытую ссылку `[[Environment]]` на текущее лексическое окружение. При вызове поиск переменной идёт по цепочке окружений наружу. Окружение живёт, пока на него ссылается хоть одна живая функция — GC не может его собрать.

```js
function createCounter() {
  let count = 0; // живёт в окружении createCounter
  return () => ++count; // замыкание удерживает окружение
}
```

**Классическая утечка:**

```js
function attach(el) {
  const bigData = loadHugeArray(); // 100 МБ
  el.addEventListener("click", () => {
    console.log(bigData.length); // замыкание держит bigData
  });
}
```

Пока элемент и обработчик живы — жив и `bigData`. Хуже: в V8 функции, созданные в одном окружении, **делят один Context-объект**, поэтому колбэк может удерживать переменную, которую сам не использует, если её использует «соседняя» функция из того же окружения.

**Как чинить:**

- Не захватывать лишнее: вычислить нужное значение заранее (`const len = bigData.length`) и замкнуть только его.
- Снимать обработчики (`removeEventListener`, `AbortController` с `signal`), особенно на долгоживущих объектах (window, сторы).
- Обнулять ссылки на большие структуры, когда они больше не нужны.
- Диагностика: DevTools → Memory → Heap Snapshot, искать Detached nodes и Retainers.

**Про `var` в цикле:** `var` создаёт одну переменную на всю функцию — все колбэки замыкаются на одно окружение и видят финальное значение. `let` в `for` создаёт **новое окружение на каждую итерацию** — каждый колбэк видит свою копию.
