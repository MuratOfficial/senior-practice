Когда микросервисы оправданы, а когда это дорогая ошибка? Что вы получаете и что платите при распиле монолита? Что такое модульный монолит?

<!-- answer -->

**Микросервисы решают организационную проблему, а не техническую:** десятки команд, которым нужно **независимо деплоить** свои части (закон Конвея). Побочно: независимое масштабирование горячих частей, изоляция сбоев, разные стеки, гранулярная безопасность.

**Цена — распределённая система:** сетевые вызовы вместо вызовов функций (латентность, частичные сбои, ретраи, таймауты, circuit breakers); **нет распределённых транзакций** — саги, идемпотентность, eventual consistency и её баги; распределённая отладка (tracing обязателен); контракты и версионирование API между командами; инфраструктура — оркестрация, service discovery, деплой-пайплайны на каждый сервис; **сложность переносится из кода в операции**. Микросервисы при команде в 8 человек = вся эта цена без выгоды (деплоиться независимо некому).

**Модульный монолит** — частый правильный ответ: один деплой, но внутри — жёсткие модули с явными границами (свои таблицы/схемы, общение через интерфейсы/события, запрет прямых импортов через границу — проверяется линтером). Даёт: простоту операций и транзакций монолита + готовые швы для будущего выреза сервисов, **когда** появится организационная необходимость. Практика: monolith first — границы модулей неизвестны заранее, а в монолите их менять дёшево, в микросервисах — мучительно.

**Как резать (когда пора):** по **бизнес-доменам** (bounded contexts из DDD): «заказы», «биллинг», «каталог» — а не по слоям («сервис БД», «сервис API» — распределённый монолит: деплой всё равно синхронный, связность осталась, сеть добавилась). Признаки правильной границы: минимум синхронных вызовов через неё, данные внутри, команда-владелец.

**Красные флаги на интервью:** «микросервисы = масштабируемость» (монолит масштабируется репликами), «начнём с микросервисов, чтобы потом не переписывать» (перепишете границы), общий домен-модель/БД между сервисами, синхронные цепочки из 5 сервисов на запрос.
