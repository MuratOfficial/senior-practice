Что такое дескрипторы и как через них устроены property, методы и classmethod? Каков порядок поиска атрибутов? Что делают метаклассы и когда они действительно нужны?

<!-- answer -->

**Дескриптор** — объект класса с методами `__get__(self, obj, objtype)` / `__set__` / `__delete__`, размещённый **в классе** (не в экземпляре). Доступ к атрибуту через точку делегируется этим методам. Data-дескриптор (есть `__set__`/`__delete__`) и non-data (только `__get__`) — различие критично для порядка поиска.

**Порядок поиска `obj.x`** (упрощённый `object.__getattribute__`):

1. **data-дескриптор** из `type(obj).__mro__` — выигрывает у всего;
2. `obj.__dict__` — обычный атрибут экземпляра;
3. **non-data-дескриптор** / атрибут класса;
4. `__getattr__` — только если ничего не нашли.

Поэтому `property` (data-дескриптор) нельзя «затенить» записью в экземпляр, а **метод — можно**: функции — non-data-дескрипторы.

**Всё «волшебное» — дескрипторы:** `function.__get__` возвращает bound method (так `self` привязывается при доступе через точку); `property` — data-дескриптор над fget/fset; `classmethod`/`staticmethod` — обёртки с своим `__get__`; `__slots__` генерирует дескрипторы для полей. ORM-поля (Django/SQLAlchemy) — дескрипторы: `User.name` в классе — объект колонки, `user.name` — значение.

Собственный дескриптор оправдан для **переиспользуемой** логики атрибутов (валидация, ленивые вычисления, единицы измерения) — `property` не переиспользуется между классами, дескриптор — да (+`__set_name__` для имени поля).

**Метакласс** — класс класса: `type` создаёт классы (`type(name, bases, ns)`), метакласс перехватывает создание (`__new__`) — можно валидировать/дополнять класс при определении: регистрация плагинов, автогенерация схем, ORM-модели, синглтоны, abc (ABCMeta). Правило здравого смысла: **90% задач метаклассов решаются проще** — `__init_subclass__` (хук в родителе на создание наследников), `__set_name__`, декораторы классов. Метакласс — когда нужно менять сам процесс создания класса или влиять на `isinstance` (`__instancecheck__`), и конфликтует при множественном наследовании с разными метаклассами.

**Senior-формулировка:** «дескрипторы — механизм, метаклассы — фабрика классов; property/методы/slots — дескрипторы, ABC/ORM — метаклассы; в прикладном коде предпочитаю `__init_subclass__` и декораторы за явность».
