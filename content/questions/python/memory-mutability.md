Объясните модель «всё — объект, переменные — ссылки». Что выведет код и почему? Как правильно копировать вложенные структуры?

```python
def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item(1))
print(add_item(2))

a = [[0] * 3] * 2
a[0][0] = 9
print(a)
```

<!-- answer -->

**Вывод: `[1]`, `[1, 2]`, `[[9, 0, 0], [9, 0, 0]]`.**

**Модель:** переменная — имя, привязанное к объекту; присваивание не копирует, а создаёт ещё одну ссылку. Передача аргументов — по ссылке на объект («call by object reference»): функция может **мутировать** переданный изменяемый объект, но переприсваивание параметра снаружи не видно.

**Ловушка 1 — изменяемый дефолт:** дефолтные значения вычисляются **один раз при определении** функции и живут в `__defaults__`. Один список делится между всеми вызовами. Идиома:

```python
def add_item(item, items=None):
    if items is None:
        items = []
```

**Ловушка 2 — `*` на списках:** `[[0]*3]*2` копирует **ссылки** на один и тот же внутренний список — обе строки «матрицы» это один объект. Правильно: `[[0]*3 for _ in range(2)]` (с числами `[0]*3` безопасно: int неизменяем).

**Изменяемость:** immutable — int, str, tuple, frozenset (мутация невозможна, «изменение» = новый объект; поэтому строковая конкатенация в цикле O(n²) — использовать `"".join`); mutable — list, dict, set, объекты. Tuple с изменяемым содержимым hashable не будет.

**Копирование:** `copy.copy`/срез/`dict(d)` — **поверхностное**: новый контейнер, но вложенные объекты общие. `copy.deepcopy` — рекурсивное (корректно обрабатывает циклы, дорогое). Часто лучший дизайн — иммутабельность: frozen dataclass, возвращать новые структуры.

**`is` vs `==`:** `is` — идентичность (один объект, сравнение id), `==` — равенство значений (`__eq__`). `is` корректен только для синглтонов: `None`, `True/False`, sentinel-объектов. Сравнение чисел/строк через `is` иногда «работает» из-за **интернирования** (кэш int −5..256, интернированные строки) — это деталь реализации, полагаться нельзя.
