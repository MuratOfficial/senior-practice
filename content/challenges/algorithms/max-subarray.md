      function maxSubArray(nums) {
        let current = nums[0];
        let best = nums[0];
        for (let i = 1; i < nums.length; i++) {
          current = Math.max(nums[i], current + nums[i]);
          best = Math.max(best, current);
        }
        return best;

Реализуйте `maxSubArray(nums)` — максимальную сумму непрерывного непустого подмассива (LeetCode 53). Классика динамического программирования и эталон для объяснения **алгоритма Кадане**.

Массив может содержать отрицательные числа; подмассив должен быть непустым.

<!-- explanation -->

Наивный перебор всех подмассивов — O(n²). Алгоритм **Кадане** решает за O(n) одним проходом, и это красивый пример «локальный оптимум → глобальный».

На каждом элементе `x` есть ровно два выбора для «лучшего подмассива, заканчивающегося здесь»:

- **продолжить** предыдущий: `current + x`;
- **начать заново** с `x`: если накопленная сумма стала обузой (ушла в минус), выгоднее стартовать с текущего элемента.

Отсюда `current = max(x, current + x)`. Параллельно тянем глобальный максимум `best = max(best, current)`.

Два подводных камня, которые проверяют тесты:

1. **Все числа отрицательные** (`[-3,-1,-2]` → `-1`). Инициализация `best = 0` дала бы неверный `0` (пустой подмассив запрещён). Поэтому инициализируем **первым элементом**, а не нулём.
2. **Инициализация с `nums[0]`** и проход с индекса 1 — единственный корректный старт для непустого требования.

Это DP с оптимизацией по памяти: классическая формулировка держит массив `dp[i]` = «лучшая сумма, кончающаяся на i», но нужен только предыдущий `dp[i-1]`, поэтому храним одну переменную `current` — O(1) память.

**Follow-up:** вернуть **индексы** подмассива (запоминать старт при рестарте); вариант с произведением (max product subarray — нужно тянуть и min из-за двух минусов = плюс); почему Кадане — это DP, а не жадность в чистом виде (выбор локально оптимален, но обоснован рекуррентой).
