    solution: |
      function mergeIntervals(intervals) {
        if (intervals.length === 0) return [];
        const sorted = [...intervals].sort((a, b) => a[0] - b[0]);
        const result = [[...sorted[0]]];

        for (let i = 1; i < sorted.length; i++) {
          const [start, end] = sorted[i];
          const last = result[result.length - 1];
          if (start <= last[1]) {
            last[1] = Math.max(last[1], end); // пересечение/касание — расширяем
          } else {
            result.push([start, end]);
          }
        }
        return result;
      }

Классика (LeetCode 56), постоянно встречается в прикладном виде: слоты календаря, диапазоны бронирования, объединение временных окон.

Дан массив интервалов `[start, end]` в произвольном порядке. Слейте все пересекающиеся (включая касающиеся границами: `[1,4]` и `[4,5]` → `[1,5]`) и верните результат, отсортированный по началу. Вход мутировать нельзя.

<!-- explanation -->

Ключевая идея: после **сортировки по началу** новый интервал может пересекаться только с **последним** накопленным — всё, что левее, уже гарантированно отделено. Это сводит задачу к одному проходу:

- `start <= last.end` → пересечение или касание: расширяем конец `last.end = max(last.end, end)` — `max` обязателен из-за вложенных интервалов (`[1,10]` и `[2,3]`: конец должен остаться 10);
- иначе — начинаем новый интервал.

Сложность: O(n log n) на сортировку + O(n) проход. Память O(n) (копия для сортировки — `sort` мутирует, а вход менять нельзя; скрытый тест это проверяет — привычка, отличающая аккуратного кандидата).

Типичные ошибки: сравнение `start < last.end` (теряется касание границами), `last[1] = end` без `max` (вложенные интервалы обрезают результат), сортировка по умолчанию (`sort()` без компаратора сортирует **как строки** — `[10, ...]` встанет раньше `[2, ...]`).

**Follow-up:** вставить один интервал в отсортированный список (LeetCode 57 — без пересортировки, O(n)); найти свободные слоты (инверсия задачи); интервальные пересечения двух списков; sweep line как обобщение (события +1/−1, максимум перекрытий — «сколько переговорок нужно»).
