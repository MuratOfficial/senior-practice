title: Реализовать once
difficulty: easy
tags: [closures, higher-order]
hints:
  - Храните флаг «уже вызывали» и закешированный результат в замыкании.
  - Последующие вызовы должны возвращать первый результат и не выполнять fn повторно.
languages:
  - id: javascript
    starter: |
      /**
       * Возвращает обёртку, которая вызывает fn только при первом вызове,
       * а дальше отдаёт закешированный результат первого вызова.
       */
      function once(fn) {
        // ваш код
      }
    solution: |
      function once(fn) {
        let called = false;
        let result;
        return function (...args) {
          if (!called) {
            called = true;
            result = fn.apply(this, args);
          }
          return result;
        };
      }
    tests:
      - name: "fn выполняется только один раз"
        code: |
          let calls = 0;
          const init = once(() => { calls++; return "ready"; });
          init(); init(); init();
          assert.equal(calls, 1);
      - name: "возвращает результат первого вызова"
        code: |
          let n = 0;
          const next = once(() => ++n);
          assert.equal(next(), 1);
          assert.equal(next(), 1, "повторный вызов отдаёт первый результат");
      - name: "аргументы берутся из первого вызова"
        code: |
          const f = once((x) => x * 10);
          assert.equal(f(2), 20);
          assert.equal(f(5), 20, "аргументы второго вызова игнорируются");
      - name: "сохраняет this"
        hidden: true
        code: |
          const obj = { v: 7, get: once(function () { return this.v; }) };
          assert.equal(obj.get(), 7);
      - name: "кеширует undefined-результат тоже"
        hidden: true
        code: |
          let calls = 0;
          const f = once(() => { calls++; });
          f(); f();
          assert.equal(calls, 1, "даже если fn вернула undefined, повторно не вызываем");
---
Реализуйте `once(fn)` — обёртку, гарантирующую, что `fn` выполнится **не более одного раза**; последующие вызовы возвращают результат первого. Разминка на замыкания, часто идёт перед `memoize`.

Требования:

- `fn` вызывается только при первом обращении;
- дальше возвращается закешированный результат первого вызова (аргументы последующих вызовов игнорируются);
- даже если `fn` вернула `undefined`, повторно не вызываем.

<!-- explanation -->

Замыкание держит два состояния: булев флаг `called` и закешированный `result`. Логика тривиальна, но есть тонкость, которую тест проверяет специально:

**Нельзя определять «вызывали ли уже» по `result !== undefined`.** Если `fn` легитимно возвращает `undefined` (инициализация без возвращаемого значения), такая проверка заставит вызывать `fn` при каждом обращении. Поэтому нужен **отдельный флаг** `called`, а не проверка результата на «пустоту». Это частая ошибка на собеседовании.

`this` и аргументы первого вызова пробрасываются через `fn.apply(this, args)` — как и в `debounce`/`memoize`.

Практический смысл: ленивая одноразовая инициализация (открыть соединение, показать баннер один раз, гарантия «конструктор побочного эффекта отработал единожды»). В lodash это `_.once`; частный случай более общего `_.before(n, fn)`.

**Follow-up:** чем `once` отличается от `memoize` (memoize кеширует **по аргументам** — разные входы дают разные результаты; once игнорирует аргументы и всегда отдаёт первый результат); как сбросить состояние (вернуть пару `[wrapped, reset]`); потокобезопасность аналога в многопоточных языках (double-checked locking, `sync.Once` в Go).
