      /**
       * curry(fn) возвращает каррированную версию fn:
       * её можно вызывать по одному аргументу за раз или группами,
       * пока не наберётся арность fn — тогда fn выполняется.
       */
      function curry(fn) {
        // ваш код
      }
    solution: |
      function curry(fn) {
        return function curried(...args) {
          if (args.length >= fn.length) {
            return fn.apply(this, args);
          }
          return function (...rest) {
            return curried.apply(this, args.concat(rest));
          };
        };
      }
    tests:

          const sum = curry((a, b, c) => a + b + c);
          assert.equal(sum(1, 2, 3), 6);
      - name: "смешанные группы"

          const sum = curry((a, b, c) => a + b + c);
          assert.equal(sum(1, 2)(3), 6);
          assert.equal(sum(1)(2, 3), 6);

Реализуйте `curry(fn)` — преобразование функции нескольких аргументов в цепочку функций одного (или нескольких) аргументов. Частый вопрос секции про замыкания и функциональный стиль.

Требования:

- `sum(1)(2)(3)`, `sum(1, 2, 3)`, `sum(1, 2)(3)` — все дают один результат;
- как только накоплено аргументов не меньше арности `fn` (`fn.length`) — `fn` выполняется;
- частичное применение переиспользуемо: `const add10 = sum(10)` не должно «портиться» между вызовами.

<!-- explanation -->

Основа — рекурсия с накоплением аргументов в замыкании. `curried` на каждом вызове сравнивает число собранных аргументов с `fn.length` (**арность** — число объявленных параметров):

- набрали достаточно → `fn.apply(this, args)`;
- не хватает → возвращаем новую функцию, которая **конкатенирует** новые аргументы с уже собранными и рекурсивно зовёт `curried`.

Ключ к переиспользуемости — `args.concat(rest)` создаёт **новый** массив, а не мутирует существующий. Поэтому `add10 = sum(10)` можно ветвить многократно: каждая ветка получает свою копию накопленного.

Подводные камни:

1. `fn.length` **не считает** параметры со значением по умолчанию и rest-параметр — для таких функций арность «ломается», каррирование нужно с явной арностью: `curry(fn, arity)`.
2. `this` пробрасывается через `apply`, если каррируемая функция используется как метод.

**Follow-up:** отличие каррирования от частичного применения (`partial` фиксирует часть аргументов и сразу возвращает функцию оставшейся арности — не обязательно по одному); зачем это на практике (point-free стиль, конфигурируемые хелперы `map(fn)`); вариант с placeholder (`_`) как в lodash.
