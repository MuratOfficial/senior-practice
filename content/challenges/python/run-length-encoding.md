    solution: |
      from itertools import groupby

      def encode(s: str) -> str:
          return "".join(f"{ch}{len(list(group))}" for ch, group in groupby(s))

      def decode(s: str) -> str:
          result = []
          i = 0
          while i < len(s):
              ch = s[i]
              i += 1
              j = i
              while j < len(s) and s[j].isdigit():
                  j += 1
              count = int(s[i:j])
              result.append(ch * count)
              i = j
          return "".join(result)

Реализуйте пару `encode`/`decode` для простейшего **run-length encoding**: серии одинаковых символов заменяются на «символ + длина». Практическая задача на группировку и аккуратный разбор строки; проверяет, что вы держите в голове **инвариант обратимости**.

- `encode("aaabbc")` → `"a3b2c1"`;
- `decode` — строгая инверсия, счётчик может быть многозначным (`a12`).

<!-- explanation -->

**Кодирование** идиоматично через `itertools.groupby`, который как раз группирует **подряд идущие** равные элементы (в отличие от SQL `GROUP BY` — важно, что только соседние). Для каждой группы берём символ и длину. Ручной вариант — проход с указателем и счётчиком текущей серии.

**Декодирование** сложнее, чем кажется, из-за многозначных счётчиков: нельзя читать «символ + одна цифра». Нужен разбор — прочитать символ, затем «съесть» **все** последующие цифры (`while isdigit`), собрать число, размножить. Именно тест `a12b1` ловит наивные решения, читающие ровно один символ счётчика.

Главное свойство, которое стоит проверять тестом, — **`decode(encode(s)) == s`** для любого входа (round-trip). Оно ловит рассогласование форматов лучше, чем отдельные примеры.

Важная оговорка про «сжатие»: этот формат **раздувает** данные без длинных серий (`abc` → `a1b1c1`, вдвое длиннее). Настоящий RLE кодирует счётчик только когда серия длиннее порога, а символы-цифры в алфавите требуют экранирования — иначе `decode` не разберёт. На собеседовании стоит это назвать: наивный RLE полезен лишь на данных с длинными сериями (например, битовые маски, простая графика).

**Follow-up:** как сделать формат самоописываемым при цифрах в данных (эскейпинг/разделитель); LeetCode 443 (сжатие «на месте» in-place с O(1) доп. памяти); связь с реальными алгоритмами (RLE как шаг в факсах/BMP, префикс к энтропийному кодированию).
