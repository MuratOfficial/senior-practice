          {"a": {"b": 1, "c": {"d": 2}}} -> {"a.b": 1, "a.c.d": 2}

    solution: |
      def flatten_dict(d: dict, sep: str = ".") -> dict:
          result = {}

          def walk(node, prefix):
              for key, value in node.items():
                  path = f"{prefix}{sep}{key}" if prefix else str(key)
                  if isinstance(value, dict) and value:
                      walk(value, path)
                  else:
                      result[path] = value

          walk(d, "")
          return result

Реализуйте `flatten_dict(d, sep=".")` — выпрямление вложенного словаря в плоский, где ключ отражает путь: `{"a": {"b": 1}}` → `{"a.b": 1}`. Практическая задача (конфиги, JSON из API, метки метрик), проверяющая рекурсию и работу со словарями.

Требования:

- вложенные словари разворачиваются рекурсивно с составным ключом через `sep`;
- невложенные значения (числа, строки, **списки**) остаются как есть — списки не разворачиваем;
- пустой вложенный словарь считается значением-листом.

<!-- explanation -->

Рекурсивный обход с накоплением префикса. Внутренняя функция `walk(node, prefix)` идёт по парам: если значение — непустой `dict`, спускаемся с обновлённым путём `prefix + sep + key`; иначе записываем лист в результат. Пустой `prefix` на верхнем уровне обрабатываем отдельно, чтобы ключ не начинался с разделителя.

Ключевые решения дизайна, которые стоит проговорить:

1. **Списки не разворачиваем** — типичное требование: `{"tags": [1, 2]}` должно остаться списком, а не превратиться в `tags.0`, `tags.1`. Проверяем строго `isinstance(value, dict)`.
2. **Пустой dict — это лист.** Условие `and value` не даёт `{}` «исчезнуть»: без него пустой словарь не попал бы ни в рекурсию (нечего обходить), ни в результат.
3. **Коллизии ключей.** Если во входе есть и `{"a.b": 1}`, и `{"a": {"b": 2}}`, плоские ключи совпадут — в реальном коде это повод выбрать разделитель, которого нет в ключах, или падать явной ошибкой.

Обратная операция (`unflatten`) — восстановление вложенности `split(sep)` по каждому ключу и спуск с `setdefault`.

**Follow-up:** генераторная версия (`yield (path, value)` вместо накопления в dict — ленивее, композируется); почему рекурсия ограничена глубиной стека Python (~1000) и когда нужен явный стек; как это делает `pandas.json_normalize`.
