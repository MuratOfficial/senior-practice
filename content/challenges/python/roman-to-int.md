      def roman_to_int(s: str) -> int:
          values = {"I": 1, "V": 5, "X": 10, "L": 50, "C": 100, "D": 500, "M": 1000}
          total = 0
          for i, ch in enumerate(s):
              if i + 1 < len(s) and values[ch] < values[s[i + 1]]:
                  total -= values[ch]
              else:
                  total += values[ch]
          return total

Реализуйте `roman_to_int(s)` — преобразование римского числа в целое (LeetCode 13). Простая, но показательная задача на разбор строки с локальным look-ahead.

Символы: `I=1, V=5, X=10, L=50, C=100, D=500, M=1000`. Вход корректен, диапазон 1..3999.

<!-- explanation -->

Обычно римская запись — сумма значений слева направо (`LVIII` = 50+5+1+1+1). Исключение — шесть **вычитающих** пар (`IV`, `IX`, `XL`, `XC`, `CD`, `CM`), где меньший символ стоит **перед** большим и вычитается.

Изящный трюк вместо разбора пар: идём посимвольно и смотрим на **следующий** символ. Правило одно — «если текущее значение **меньше** следующего, вычитаем его, иначе прибавляем». Это автоматически покрывает все шесть пар без их перечисления: в `IX` единица меньше десятки → `-1`, потом `+10` = 9.

Почему это корректно: в валидной записи символ меньше соседа справа **только** в вычитающей позиции — в остальных случаях цифры не возрастают. Одна проверка `values[ch] < values[next]` ловит ровно нужные случаи.

Сложность O(n) время, O(1) память (словарь фиксированного размера). Альтернатива — заменить пары строкой (`s.replace("IV","IIII")…`) и просуммировать, но look-ahead чище и не создаёт временных строк.

**Follow-up:** обратная задача `int_to_roman` (жадно вычитаем крупнейшие номиналы из таблицы, включая пары `900→CM`); как **валидировать** римское число (не более трёх одинаковых подряд, корректные пары) — регулярное выражение или конечный автомат; почему диапазон ограничен 3999 (нет символа для 5000+ в базовой нотации).
